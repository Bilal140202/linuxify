# Linuxify вЂ” Shared Project Context (For Documentation Agents)

> **Read this file in full before writing any documentation.** All docs must be consistent with the facts, terminology, and vision defined here.

## 1. What is Linuxify?

**Linuxify** is an open-source developer toolkit that lets you run **Linux-oriented developer CLIs** (such as `cline`, `codex`, `aider`, `goose`, `gemini-cli`, `openhands`, `freebuff`, and similar tools) **directly on Android** with a single command, by automatically setting up and managing a Ubuntu (or other distro) `proot` environment inside Termux.

The tagline candidates:

- *"Run Linux developer tools on Android."*
- *"Desktop developer tools. Android simplicity."*
- *"Linux CLIs. Zero hassle."*

The brand "Linuxify" is intentionally verb-like (`linuxify install`, `linuxify add cline`, `linuxify doctor`) in the spirit of `git`, `npm`, `cargo` вЂ” a first-class developer tool, not a wrapper script.

## 2. The Problem

Today, running modern AI/developer CLIs on Android requires a long manual ritual:

```
Install Termux
  в†’ Install proot
    в†’ Install Ubuntu
      в†’ Install Node
        в†’ Fix PATH
          в†’ Figure out why process.platform=android
            в†’ Patch the CLI source
              в†’ Finally use the tool
```

This is repeated for **every** CLI (Cline, Codex, Freebuff, Aider, etc.). Each tool fails in similar ways: it assumes `process.platform === "linux"`, expects glibc, expects desktop environment variables, hardcodes `x86_64`, etc. The pain is **systemic, not tool-specific**.

## 3. The Solution

With Linuxify:

```bash
pkg install linuxify        # or curl install
linuxify init               # set up Ubuntu + runtimes + PATH
linuxify add cline          # install + patch + launcher
cline                       # works directly from Termux shell
```

Linuxify is a **compatibility layer + package manager + diagnostic engine** for Linux developer tooling on Android. It is *not* "another Termux script."

## 5. Command Surface (v1)

```
linuxify init                # Bootstrap environment (idempotent)
linuxify install             # Alias for init in interactive mode

```

  - file: "node_modules/cline/dist/arch.js"
    find: "process.arch === 'x64'"
    replace: "['x64','arm64'].includes(process.arch)"

  CLINE_PLATFORM: linux
  FORCE_COLOR: "1"

  min_linuxify: "0.1.0"
  tested_distros: [ubuntu, debian]
  known_issues: []

doctor:
  - check: node_version
    min: 20
  - check: executable
    binary: cline
```

## 8. Target Users

- **Primary**: Developers using Android phones/tablets (Termux) who want to run AI coding agents and other Linux CLIs.
- **Secondary**: Students, hobbyists, road-warrior engineers, contributors to the open-source AI ecosystem.
- **Tertiary**: Future cloud users (when registry/cloud sync lands).

## 9. Target Platforms

- **Host**: Android 9+ via Termux (from F-Droid, *not* Play Store).
- **Architectures**: `aarch64` (primary), `armv7l` (best-effort), `x86_64` (Chromebook/Android-x86).
- **Distros (v1)**: Ubuntu 24.04 default. Pluggable: Debian, Arch, Alpine.
- **Runtimes (v1)**: Node.js LTS, Python 3.12, Git, common build tools (build-essential, pkg-config, etc.).

## 10. Non-Goals (v1)

- Not a general-purpose Linux distribution.
- Not a root-required tool (works without root).
- Not a replacement for Termux вЂ” it sits on top of Termux.
- Not an emulator вЂ” uses `proot` for syscall translation.
- Not a GUI toolkit вЂ” CLI-first.
- Not a cloud service (v1) вЂ” fully local. Cloud features are future work.

## 13. Technology Choices (proposed, document as ADRs)

- **CLI core**: TypeScript + Node.js (so Linuxify can run inside its own Ubuntu proot and use its own Node runtime).
- **Package definitions**: YAML (human-editable, contributable, schema-validated).
- **Distro backend**: `proot-distro` as the default; pluggable.
- **Launcher mechanism**: Symlinked shell scripts in `$PREFIX/bin/` that exec into proot.
- **Patcher**: AST-aware (for JS/TS) + regex fallback (for everything else).
- **Doctor**: Declarative check list per component, each check returns `ok | warn | fail | missing` + remediation hint.
- **Config store**: Single `~/.linuxify/config.toml` + per-dist `~/.linuxify/state.json`.
- **Storage layout**: `~/.linuxify/` is the home for everything (distros, runtimes, packages, state, logs, cache).
